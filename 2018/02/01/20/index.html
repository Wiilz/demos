发表于 2018-02-01 | 分类于 其他

触发器trigger：

触发器也是一段预先定义好的代码，并有个名字，但是它不可以被手动调用，而是在某个表发生某个事件的时候被自动调用； 定义形式：

create triggle 触发器名 出发时机 触发时间 on 表名 for each row as begin //编程语句 end;

说明：

  • 出发时机只有两个，before，after
  • 出发时间，只有3个 inser，update ，delete
  • 即 触发器的含义是： 在某个表进行insert（或update 或delete）之前或之后，会去执行其中写好的代码；因此每个表只有6个情形可以被调用。一般没人写6个；
  • 通常，触发器用于在对某个表进行增删改的同时又需要进行其他操作时使用
  • 在触发器的内部，有两个特殊的用来获取数据的关键字：
    1. new： 代表当前正要执行的insert或update时候的‘新行’数据；通过它可以获得这一数据的任意一个字段的值： 比如： set @v1=new.id#获取这个新行的id值；
    2. old：代表当前正要执行的delete时候的‘旧行’数据；比如： set @v2=old.age;
